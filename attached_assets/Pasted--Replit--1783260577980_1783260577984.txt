الجزء 1 — دعم كل مزوّدي الذكاء الاصطناعي:


وسّع القاموس MODELS والقائمة OR_MODELS (قرب السطر 15–30) لإضافة مزوّدين جدد، كل نموذج يحمل حقل provider واسم المفتاح المطلوب:

Anthropic Claude (claude-3.5-sonnet, claude-3.5-haiku) — https://api.anthropic.com/v1/messages
Google Gemini (gemini-2.0-flash, gemini-1.5-pro) — Google Generative Language API
DeepSeek (deepseek-chat, deepseek-reasoner) — https://api.deepseek.com
Mistral — https://api.mistral.ai/v1
Groq — https://api.groq.com/openai/v1
xAI Grok — https://api.x.ai/v1
Together AI — https://api.together.xyz/v1
أبقِ OpenRouter و g4f كما هما.

عمّم الدالة _or_stream (السطور 114–158) إلى دالة عامة _openai_compat_stream(messages, model_name, key, base_url) تعمل مع كل المزوّدين المتوافقين مع صيغة OpenAI (OpenAI, DeepSeek, Mistral, Groq, xAI, Together, OpenRouter).
أضف _anthropic_stream(messages, model_name, key) لصيغة Anthropic (حوّل رسالة system إلى حقل system منفصل، واستخدم ترويسة x-api-key + anthropic-version).

أضف دالة _get_provider_key(provider) تقرأ من _load_local_secrets() ثم من متغيرات البيئة، على نمط _get_key() (السطور 73–86) و_gh_token() (السطور 88–101) الحاليَين. المفاتيح: openai_key, anthropic_key, gemini_key, deepseek_key, mistral_key, groq_key, xai_key, together_key (بالإضافة إلى openrouter_key وgithub_token الموجودَين).

الجزء 2 — تحويله إلى وكيل برمجي مستقل:


أضف مساراً جديداً POST /admin/code-assistant/agent/stream محمياً بـ @login_required و@_super_admin_required، يشغّل حلقة tool-calling:


عرّف الأدوات: read_file, list_files, search_code, write_file, create_file, analyze_tree.
نفّذ الأدوات بإعادة استخدام _gh_read, _gh_list, _gh_write, _gh_tree الموجودة، وأضف _gh_search (بحث نصي داخل الشجرة أو GitHub Search API).
كرّر الحلقة حتى ينتهي النموذج (حد أقصى ~15 تكراراً)، وابثّ كل خطوة عبر SSE بأحداث: {"tool":...,"args":...}, {"tool_result":...}, {"chunk":...}.
عند write_file/create_file ارفع مباشرة إلى GitHub دون انتظار المستخدم؛ وعند خطأ HTTP 409 أعد جلب sha عبر _gh_read وأعد المحاولة مرة واحدة.
عدّل _build_system (يبدأ عند السطر 266) بتعليمات وكيل واضحة (استكشف واقرأ وعدّل وارفع بنفسك)، وزد حدود قصّ السياق ([:3000] وopen_files[:4]).
النماذج التي لا تدعم tool-calling (g4f) استخدم معها آلية الوسوم النصية [OPEN_FILE]/[APPLY_TO] الحالية كبديل.
في admin_code_assistant.html:


عدّل loadRepo() (السطور 478–500) بحيث يستدعي /admin/code-assistant/analyze تلقائياً بعد جلب الشجرة ويعبّئ state.projectMap، مع مؤشّر "جارٍ قراءة وتحليل المستودع".
عدّل sendMsg() لاستخدام agent/stream (أو أضف مبدّلاً بين "دردشة" و"وكيل مستقل").
اعرض خطوات الوكيل كبطاقات (📖 قراءة، 🔍 بحث، ✏️ تعديل ورفع) عند وصول أحداث tool/tool_result، وحدّث شجرة الملفات وتبويبات المحرر تلقائياً بعد كل عملية كتابة (استدعِ loadFiles/reloadTab).

وسّع قائمة modelSelect (المبنية من MODELS) لعرض كل النماذج مجمّعة حسب المزوّد.
